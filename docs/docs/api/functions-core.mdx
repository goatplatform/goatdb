# Core Functions

## itemPath

**<code>itemPath(type: T, repo: string, item: string, embed: string): string</code>**

Given the path parts, composes them into a GoatDB path.

## itemPathGetPart

**<code>itemPathGetPart(path: string, part: any | any | any | any): T</code>**

Given a DB path and a path component, this function returns the relevant
portion of the path.

## itemPathGetRepoId

**<code>itemPathGetRepoId(path: string): string</code>**

Given a DB path, return just the repository prefix of it.

## itemPathIsValid

**<code>itemPathIsValid(path: string): boolean</code>**

Checks if the given path is valid.
Valid paths must contain lowercase letters, numbers, and the characters
`-` and `_`. They must also contain at most 4 components separated by `/`
characters.

## itemPathJoin

**<code>itemPathJoin(prefix: string, suffix: string): string</code>**

Joins the given prefix and suffix into a single path.

## itemPathNormalize

**<code>itemPathNormalize(path: string): string</code>**

Normalizes the given path ensuring it has the correct number of parts
and no trailing separator.

## uniqueId

**<code>uniqueId(length: number): string</code>**

Generates a random ID string.

The generated ID is URL-safe, using only lowercase letters and numbers.
The default length of 24 characters provides a large enough space to
ensure a very low probability of collisions for most applications.

Note: This function uses Math.random() which is not cryptographically secure.
Do not use for security-sensitive purposes.

With 36 possible characters (a-z, 0-9) and a length of 24, there are
36^24 (approximately 2.24 * 10^37) possible unique IDs, making
collisions extremely unlikely for most use cases.
